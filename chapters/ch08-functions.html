ฟังก์ชัน

เรียนรู้การจัดระเบียบโค้ดด้วยฟังก์ชัน ตั้งแต่การนิยาม การส่งพารามิเตอร์ การคืนค่า ไปจนถึงขอบเขตของตัวแปร — เครื่องมือที่ขาดไม่ได้ในการเขียนโปรแกรมสมัยใหม่

CLO3: สามารถออกแบบและนำฟังก์ชันมาใช้แก้ปัญหาได้อย่างถูกต้อง

⚙️ 1. ฟังก์ชันคืออะไร?

ฟังก์ชัน (Function) คือกลุ่มของคำสั่งที่ถูกจัดรวมกันไว้เพื่อทำงานอย่างใดอย่างหนึ่ง สามารถเรียกใช้ซ้ำได้หลายครั้งโดยไม่ต้องเขียนโค้ดซ้ำ

หลักการสำคัญที่ฟังก์ชันช่วยแก้คือ DRY (Don't Repeat Yourself) — ถ้าต้องเขียนโค้ดเดิมซ้ำกันมากกว่า 2 ครั้ง ให้พิจารณาสร้างเป็นฟังก์ชัน

  • ใช้ซ้ำได้ (Reusability) — เขียนครั้งเดียว เรียกได้หลายที่
  • อ่านง่ายขึ้น — แยกโค้ดซับซ้อนออกเป็นส่วนย่อย ๆ ที่มีชื่อ
  • แก้ไขง่าย — เปลี่ยนการทำงานในที่เดียว มีผลทุกที่ที่เรียกใช้
  • ทดสอบได้ — ทดสอบแต่ละฟังก์ชันแยกกันได้
โครงสร้างของฟังก์ชัน Python
  • def — keyword สำหรับนิยามฟังก์ชัน
  • function_name — ชื่อฟังก์ชัน (ใช้ snake_case)
  • (parameters) — รายการพารามิเตอร์ (ถ้าไม่มีให้ว่างไว้)
  • : — จบบรรทัด def ด้วยเครื่องหมาย colon เสมอ
  • Body — ถัดบรรทัดถัดไปต้องย่อหน้าเข้า (indent) 4 ช่องว่าง
  • return — คืนค่ากลับ (ถ้าไม่มี จะคืน None)
Python 
# โครงสร้างฟังก์ชันพื้นฐาน
def greet():
    print("สวัสดีครับ!")
    print("ยินดีต้อนรับสู่บทเรียน Python")

# การเรียกใช้ฟังก์ชัน
greet()      # เรียกครั้งที่ 1
greet()      # เรียกครั้งที่ 2 — ไม่ต้องเขียนโค้ดซ้ำ

# เปรียบเทียบ: โค้ดแบบไม่มีฟังก์ชัน (ไม่ดี)

# print("ยินดีต้อนรับสู่บทเรียน Python")

📥 2. Parameters และ Arguments

ฟังก์ชันรับข้อมูลจากภายนอกผ่าน พารามิเตอร์ (Parameter) และ อาร์กิวเมนต์ (Argument)

  • Parameter — ชื่อตัวแปรในวงเล็บตอนนิยามฟังก์ชัน เช่น def add(a, b) → a และ b คือ parameter
  • Argument — ค่าจริงที่ส่งไปตอนเรียกใช้ เช่น add(3, 5) → 3 และ 5 คือ argument

การส่ง argument มี 2 แบบหลัก:

  • Positional Arguments — ส่งตามลำดับ เช่น add(3, 5)
  • Keyword Arguments — ระบุชื่อพารามิเตอร์ เช่น add(b=5, a=3) — ลำดับไม่สำคัญ
Python 
# ฟังก์ชันที่มี parameters
def greet(name, greeting):
    print(f"{greeting}, {name}!")

# Positional arguments — ส่งตามลำดับ
greet("สมชาย", "สวัสดี")
greet("Python", "Hello")

# Keyword arguments — ระบุชื่อ, ลำดับไม่สำคัญ
greet(greeting="Hi", name="World")

# ฟังก์ชันคำนวณ
def calculate_area(width, height):
    area = width * height
    print(f"พื้นที่ = {width} x {height} = {area}")

calculate_area(5, 3)
calculate_area(height=10, width=4)  # keyword args

📤 3. Return Values (การคืนค่า)

ฟังก์ชันสามารถคืนค่ากลับไปให้ผู้เรียกใช้ด้วยคำสั่ง return ซึ่งทำให้สามารถนำผลลัพธ์ไปใช้ต่อได้

  • ถ้าไม่มี return หรือ return โดยไม่มีค่า — ฟังก์ชันคืนค่า None
  • คืนค่าได้หลายค่าพร้อมกันโดยใช้ Tuple return a, b
  • เมื่อโปรแกรมถึง return จะหยุดทำงานและออกจากฟังก์ชันทันที
Python 
# ฟังก์ชันที่คืนค่าเดียว
def square(n):
    return n ** 2

result = square(5)
print(f"5^2 = {result}")       # 25
print(f"7^2 = {square(7)}")    # 49

# ฟังก์ชันคืนหลายค่า (Tuple)
def min_max(numbers):
    return min(numbers), max(numbers)

low, high = min_max([85, 42, 98, 67, 73])
print(f"ต่ำสุด: {low}, สูงสุด: {high}")

# ฟังก์ชันที่ไม่มี return (คืน None)
def say_hello(name):
    print(f"Hello, {name}!")

val = say_hello("World")
print(f"ค่าที่ได้กลับมา: {val}")  # None

# ใช้ return เพื่อออกจากฟังก์ชันเร็ว
def check_positive(n):
    if n <= 0:
        return "ต้องเป็นจำนวนบวก"
    return f"{n} เป็นจำนวนบวก"

print(check_positive(5))
print(check_positive(-3))

🌐 4. Scope (ขอบเขตตัวแปร)

Scope คือขอบเขตที่ตัวแปรสามารถถูกเข้าถึงได้ Python มีกฎ LEGB ในการค้นหาตัวแปร

  • Local — ตัวแปรที่สร้างในฟังก์ชัน ใช้ได้เฉพาะในฟังก์ชันนั้น
  • Enclosing — ตัวแปรในฟังก์ชันที่ครอบอยู่ (nested function)
  • Global — ตัวแปรที่สร้างนอกฟังก์ชันทั้งหมด
  • Built-in — ชื่อที่ Python กำหนดไว้แล้ว เช่น print, len
Local vs Global Variable
ประเภทที่ประกาศใช้ได้ที่ไหน
Localในฟังก์ชันเฉพาะในฟังก์ชันนั้น
Globalนอกฟังก์ชันทุกที่ในไฟล์
Python 
x = 10  # Global variable

def show_scope():
    y = 20  # Local variable
    print(f"ในฟังก์ชัน: x = {x}")  # อ่าน global ได้
    print(f"ในฟังก์ชัน: y = {y}")  # อ่าน local ได้

show_scope()
print(f"นอกฟังก์ชัน: x = {x}")   # ใช้ได้
# print(y)  # NameError! y อยู่ใน scope ของฟังก์ชันเท่านั้น

# ใช้ global keyword เพื่อแก้ไข global variable ในฟังก์ชัน
count = 0

def increment():
    global count    # บอก Python ว่าจะใช้ count ระดับ global
    count += 1
    print(f"count = {count}")

increment()  # count = 1
increment()  # count = 2
increment()  # count = 3
print(f"ค่าสุดท้าย: {count}")
แนวทางที่ดีกว่าการใช้ global

การใช้ global มากเกินไปทำให้โค้ดติดตามยาก ควรใช้ return value แทนเมื่อทำได้:

# แบบแนะนำ: ใช้ return แทน global
def increment(count):
    return count + 1

count = 0
count = increment(count)  # count = 1
count = increment(count)  # count = 2

🎛️ 5. Default Arguments (ค่าเริ่มต้น)

Default Arguments คือค่าที่กำหนดไว้ล่วงหน้าสำหรับพารามิเตอร์ ถ้าผู้เรียกไม่ส่งค่ามา ฟังก์ชันจะใช้ค่า default แทน

รูปแบบ: def func(param=default_value):

ข้อควรระวัง: พารามิเตอร์ที่มีค่า default ต้องอยู่ หลัง พารามิเตอร์ที่ไม่มีค่า default เสมอ

Python 
# Default argument
def greet(name, greeting="สวัสดี", punctuation="!"):
    print(f"{greeting}, {name}{punctuation}")

greet("สมชาย")                        # ใช้ค่า default ทั้งหมด
greet("วิมล", "หวัดดี")               # เปลี่ยน greeting
greet("Python", "Hello", ".")          # เปลี่ยนทั้งหมด

# ฟังก์ชัน power พร้อม default exponent
def power(base, exponent=2):
    return base ** exponent

print(power(5))      # 5^2 = 25 (ใช้ค่า default)
print(power(2, 10))  # 2^10 = 1024

# ระวัง: ห้ามใช้ mutable object เป็น default argument!
# def add_item(item, lst=[]):  # ไม่ดี! ใช้ร่วมกันทุก call
#     lst.append(item)
#     return lst

# วิธีที่ถูกต้อง: ใช้ None แล้วสร้าง list ใหม่ในฟังก์ชัน
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

print(add_item("แอปเปิล"))       # ['แอปเปิล']
print(add_item("กล้วย"))          # ['กล้วย'] — list ใหม่ ไม่ปนกัน
